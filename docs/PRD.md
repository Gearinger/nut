# 此时此地 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
- **中文名**：此时此地
- **英文名**：Cicada（待定，蝉寓意"同一时空的连接"）
- **项目代号**：Nut（开发代号）

### 1.2 产品愿景
利用 GIS + AI 技术，构建一个基于真实地理空间的、转瞬即逝的即时连接平台。让社交回归真实的物理空间与"此时此刻"。

### 1.3 产品定位
- **独立 App**：专注于空间沉浸感与底层地理能力调用
- **物理场域的过滤器**：专注于特定时间、特定物理空间内的社交，降低高噪音、低质量的社交流量
- **垂直场景的连接器**：致力于解决特定物理领域内的高效互动与信息共享

## 2. 核心功能

### 2.1 空间围栏 (Geofencing)

| 功能 | 描述 |
|------|------|
| 创建围栏 | 基于信用分的认证体系，高信用用户与官方机构可创建 |
| 准入机制 | 需要信用分门槛或密码进入 |
| 去重机制 | 空间冲突检测算法，禁止覆盖已有围栏（IoU检测） |
| 临时性 | 围栏具有明确的时间边界（活动/赛事结束即失效） |

### 2.2 即时群组与私密性

| 功能 | 描述 |
|------|------|
| 地理绑定 | 必须在围栏半径内才可加入和查看信息 |
| 地理销毁 | 离开围栏范围，群组内容自动脱敏或置灰 |
| 身份认证 | 可选的区域群组密码，增加私密性和趣味性 |

### 2.3 空间流 (Spatial Timeline)

| 功能 | 描述 |
|------|------|
| 实时流 | 围栏内的即时通讯记录 |
| 空间回溯 | 离开围栏后仍可限时查看历史帖子（不可交互） |
| 直接连接 | 可直接发起临时私聊给帖子发布者 |

## 3. 用户价值

### 3.1 核心场景

1. **活动/赛事**：漫展、演唱会、体育赛事等临时聚集场景
2. **景区**：旅游景区内的即时信息共享
3. **社区**：社区活动、邻里互助
4. **商业**：商场促销、门店活动

### 3.2 用户痛点

| 痛点 | 解决方案 |
|------|----------|
| 社交噪音大 | 地理围栏过滤，只看"附近"的内容 |
| 信息无时效性 | 围栏有时效，过期即焚 |
| 无法找到"同好" | 空间内直接匹配兴趣 |

## 4. 技术架构

### 4.1 核心技术

- **GIS**：高精度定位（GPS + WiFi + 蓝牙信标）
- **AI**：内容审核、空间意图识别、匹配推荐
- **即时通讯**：WebSocket 实时消息
- **后端**：Supabase（PostgreSQL + Realtime）

### 4.2 数据模型（Supabase）

```
users
- id, openid, username, avatar, credit_score, created_at

fences
- id, creator_id, name, center_lat, center_lng, radius, start_time, end_time, is_public, password

posts
- id, fence_id, user_id, content, media_urls, created_at, expires_at

messages
- id, fence_id, user_id, content, created_at

chats
- id, post_id, user_id_1, user_id_2, created_at
```

## 5. 商业模式

### 5.1 B端收入
- 大型漫展、景区、赛事方的围栏服务定制
- 垂直主题广告投放

### 5.2 C端收入
- 增值服务（浪漫寻人高亮、特殊推送）
- 会员订阅（更多功能、更高信用分）

### 5.3 传播策略
- 强调"相遇的珍贵感"
- 用户自发分享"此时此地相遇的故事"
